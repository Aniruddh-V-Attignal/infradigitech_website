# SEO Guide

The site ships with sensible SEO defaults. Use this guide to fine-tune.

---

## 1 · Per-page metadata

Every page exports a `metadata` object using Next.js's metadata API:

```js
export const metadata = {
  title: "Forex Display Board Manufacturer | Infra DigiTech",
  description: "…",
  keywords: ["forex display board", "currency rate board"],
  alternates: { canonical: "/forex-display-board" },
};
```

For products, edit the `seo` field on each product in `src/data/products.js`:

```js
seo: {
  title: "Forex Display Board Manufacturer in India | Infra DigiTech",
  description: "Buy LED forex display boards for banks, airports and money changers…",
  keywords: ["forex display board", "currency rate board"],
}
```

---

## 2 · Open Graph & Twitter cards

Defined globally in `src/app/layout.js` and overridden per page where needed. Replace `siteConfig.ogImage` (`/logo.svg`) with a 1200×630 PNG for rich social previews.

---

## 3 · Sitemap

Auto-generated by Next.js at `/sitemap.xml` from `src/app/sitemap.js`. It includes:

- All static pages (Home, About, Products, …)
- Every dedicated SEO product route (`/forex-display-board`, `/nurse-calling-system`, …)
- Every generic product page (`/products/<slug>`)

After adding a new dedicated SEO slug, update `seoSlugs` in `sitemap.js`.

Submit your sitemap to:

- **Google Search Console** — add property → upload `https://www.infradigitech.in/sitemap.xml`
- **Bing Webmaster Tools** — same flow

---

## 4 · `robots.txt`

Auto-generated at `/robots.txt` from `src/app/robots.js`. Allows everything, points to the sitemap.

---

## 5 · Structured data (JSON-LD)

Rendered globally in the layout:

- `Organization` — company info, social profiles
- `WebSite` — site search action

On product pages we additionally render:

- `Product` — title, description, image, brand, manufacturer
- `BreadcrumbList`

This lets Google show enhanced search results.

Test with:

- <https://search.google.com/test/rich-results>
- <https://validator.schema.org>

---

## 6 · URL structure

| Page          | URL                                               |
| ------------- | ------------------------------------------------- |
| Home          | `/`                                               |
| About         | `/about`                                          |
| All products  | `/products`                                       |
| Generic prod. | `/products/<slug>`                                |
| SEO landing   | `/<slug>` (forex-display-board, nurse-calling-system, …) |

The dedicated `/<slug>` routes exist for high-intent SEO terms — they outrank `/products/<slug>` because they're shorter and self-contained.

## 7 · Google Search Console & Google Analytics

See [`ANALYTICS_GUIDE.md`](ANALYTICS_GUIDE.md) for setup. The two main steps:

1. Add your GA4 ID to `NEXT_PUBLIC_GA_ID`.
2. Verify your domain in Search Console → submit sitemap.

---

## 8 · Image alt text

All product images use the product title as alt text — descriptive and SEO-friendly. To override, change the `alt` argument in `ProductDetail.jsx`.

---

## 9 · Content tips

- Keep H1 unique per page (already enforced — only one `h1` element per page).
- Aim for 200–400 words of meaningful copy per product (we already provide a strong base).
- Include sector-specific keywords ("bank", "hospital", "airport") in headings and body where relevant.
- Internal-link generously between related products and category pages — already done via the related-products section.
- Add real photos as soon as available — image search drives qualified B2B traffic.

---
